| Service | Image | Port |

| ldoce5-api | `ghcr.io/blue126/ldoce5-api:latest` | 5050 |
| anki-writer | `ghcr.io/blue126/anki-writer:latest` | 5051 |

Images are built for `linux/amd64` and `linux/arm64` via GitHub Actions on every push to `main`.

## Services

### ldoce5-api

Reads the LDOCE5 `.mdx` / `.mdd` files and exposes three endpoints:

| Endpoint | Description |
|----------|-------------|
| `GET /lookup?word=<w>[&sentence=<s>]` | Returns all senses, IPA pronunciation, audio URLs. With `sentence`, uses an LLM to select the best-matching sense. |
| `GET /audio/<filename>.mp3` | Streams the pronunciation audio (Speex ‚Üí MP3 via ffmpeg). |
| `GET /health` | Returns MDX/MDD load status and LLM availability. |

**`/lookup` response (abbreviated):**
```json
{
  "word": "ephemeral",
  "pronunciation": "/…™Ààfem…ôr…ôl/",
  "audio": {
    "bre": {"filename": "GB_ephemeral0205.mp3", "url": "/audio/GB_ephemeral0205.mp3"},
    "ame": null
  },
  "senses": [
    {
      "index": 0,
      "pos": "adjective",
      "definition": "lasting for only a short time",
      "definition_html": "...",
      "example": "Fashion is by nature ephemeral.",
      "gram": "",
      "freq": ["AWL"],
      "ai_selected": true
    }
  ],
  "selected_sense_index": 0,
  "warning": null
}
```

### anki-writer

Orchestrates the full card-creation flow:

1. Calls `ldoce5-api /lookup` to fetch definition + pronunciation + audio
2. Applies AI-selected (or caller-specified) sense
3. Fetches MP3 audio from `ldoce5-api /audio/...`
4. Opens the local Anki collection, syncs from the AnkiSync server, adds the note, syncs back

| Endpoint | Description |
|----------|-------------|
| `POST /add-word` | Create a card. Body: `{"word": "...", "sentence": "...", "sense_index": null}` |
| `GET /health` | Collection accessibility + ldoce5-api reachability + last sync time |

**`POST /add-word` request:**
```json
{
  "word": "ephemeral",
  "sentence": "Social media posts are often ephemeral.",
  "sense_index": null
}
```

**Note type fields written:** `word`, `pronunciation`, `definition` (HTML), `sentence`, `audio`, `extrainfo`, `url`

## Data files (not in image)

These must be present on the host and mounted as Docker volumes:

| File | Example mount path | Notes |
|------|--------------------|-------|
| LDOCE5 `.mdx` | `/data/dict/Longman Dictionary Of Contemporary English 5th.mdx` | Large, copyrighted |
| LDOCE5 `.mdd` | `/data/dict/Longman Dictionary Of Contemporary English 5th.mdd` | Audio data |
| `collection.anki2` | `/data/anki/collection.anki2` | Read/write |

> Both `.mdx` and `.mdd` require a pre-built SQLite index (`.mdx.db` / `.mdd.db`) generated by `mdict-utils`. Run `mdx -x <file>.mdx` once to create it.

## Environment variables

Copy `.env.example` as a starting point.

### ldoce5-api

| Variable | Required | Default | Description |
|----------|----------|---------|-------------|
| `LDOCE5_MDX_PATH` | yes | ‚Äî | Absolute path to `.mdx` on host |
| `LDOCE5_MDD_PATH` | no | ‚Äî | Absolute path to `.mdd` audio file |
| `API_HOST` | no | `0.0.0.0` | Bind address |
| `API_PORT` | no | `5050` | Listen port |
| `LLM_API_KEY` | no | ‚Äî | Leave empty to disable AI sense disambiguation |
| `LLM_BASE_URL` | no | `https://integrate.api.nvidia.com/v1` | OpenAI-compatible endpoint |
| `LLM_MODEL` | no | `nvidia/llama-3.1-nemotron-70b-instruct` | Model ID |

### anki-writer

| Variable | Required | Default | Description |
|----------|----------|---------|-------------|
| `COLLECTION_PATH` | yes | ‚Äî | Absolute path to `collection.anki2` on host |
| `ANKI_SYNC_URL` | no | `http://localhost:8080/` | AnkiSync server URL |
| `ANKI_SYNC_USER` | no | `anki` | Sync username |
| `ANKI_SYNC_PASSWORD` | no | `anki` | Sync password ‚Äî **use a secret vault in production** |
| `LDOCE5_API_URL` | no | `http://localhost:5050` | ldoce5-api base URL |
| `DECK_NAME` | no | `ODH` | Target Anki deck |
| `NOTE_TYPE_NAME` | no | `ODH` | Anki note type name |
| `API_HOST` | no | `0.0.0.0` | Bind address |
| `API_PORT` | no | `5051` | Listen port |

## Development

```bash
python -m venv .venv && source .venv/bin/activate
pip install -r ldoce5-api/requirements.txt
pip install -r anki-writer/requirements.txt

# Run ldoce5-api locally
cd ldoce5-api && uvicorn main:app --reload --port 5050

# Run anki-writer locally
cd anki-writer && uvicorn anki_writer:app --reload --port 5051
```

### Tests

```bash
pytest ldoce5-api/tests/
pytest anki-writer/tests/
```
